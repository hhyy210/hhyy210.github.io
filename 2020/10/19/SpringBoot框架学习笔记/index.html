SpringBoot框架学习笔记

无标签
发布日期:   2020-10-19

必学框架：springcloud、springboot、redis缓存、rabbitMq、zookeeper
一、简介
Spring Boot 框架Spring简化应用开发，之前的j2EE要结合mybatise+springMVC+spring，到一对jar包，配置一堆的文件，配置环境，SpringBoot把
spring的技术整合使用，spring data数据访问、spring framework框架、spring cloud分布式、spring security安全权限限制等等一大堆，通过整合spring的
整个系列技术站来简化企业级开发，自动配置spring的环境。
j2EE开发的一站式解决方案

二、微服务
架构风格，一种小型服务，可以通过http方式进行互通：
单体应用（所有代码都在一个里面）

app= springcloud + springboot + spring data

13、pom.xml 里面帮我们导入了很多jar包（在External Libraries文件下面）
1）spring-boot-starter-parent//Spring Boot的版本仲裁中心，以后我们导入依赖默认是不需要写版本（没有在里面管理的就需要写版本）
2）spring-boot-starter-web//spring-boot场景启动器，帮我们导入了web模块正常运行所依赖的组件

Spring Boot将所有的功能都抽取出来，做成一个个的Starters（启动器），只需要在项目里面引入这些starter相关场景的所有依赖都会导入进来，要用什么功能都导入什么场景的启动器。

14、//@SpringBootApplication 来标注一个主程序类，说明这是一个Spring Boot应用

@SpringBootApplication
public class Helloword {
        public static void main(String[] args){
            //Spring应用启动起来
            SpringApplication.run(Helloword.class,args);
        }
}
@SpringBootApplication 来标注在哪一个类上，就表明这是一个配置类（也是一个组件）

里面的@AutoConfigurationPackage：自动配置包
    将主配置类（@SpringBootApplication标注的类）的所有包及下面所有组件扫描到Spring容器；15、快速创建项目（里面可以选择功能模块，连网（可以在maven project里面刷新））

17、resources结构
static：保存所有的静态资源：js、css、images
templates：保存所有的模板页面；（Spring Boot默认jar包使用嵌入式的Tomcat，默认不支持JSP页面），可以使用模板引擎（freemarker、thymeleaf）
application.properties：spring boot应用的配置文件，可以修改一些默认文件

18、配置文件（写法不一样）
application.properties(server.port= 8088)
application.yml（属性和值大小写敏感）
配置文件的作用：修改SpringBoot自动配置的默认值；SpringBoot在底层都给我们自动配置好
yml（YAML）是一个标记语言，以数据为中心，比json、xml等更适合做配置文件
标记语言：
以前的配置文件:大多数使用的是XXX.xml文件
yml：
server:
（空格）port:(空格)8088 //以空格控制层级关系和隔阂键值

XML：
    <server>
        <port>8081</port>
    </server>

值的写法：
    字符串不用引号，用双引号表示不会转译，单引号表示会转译。
    行内写法：friend：{lastName：zhangh，age：18}
    数组：
        pets：
            - cat
            - dog

    行内写法：pets：[cat,dog]

20、@value和ConfigurationProperties,配置文件值注入：配置文件yml还是properties他们都能获取到值
松散绑定：person.firstName==person.first-name==person.first_name==PERSON_FIRST_NAME

                    ConfigurationProperties       @value                        
功能                批量注入配置文件中的属性      一个个指定
    松散绑定            支持                          不支持
SpEL（表达式）      不支持                        支持
    JSR303数据校验      支持                          不支持
复杂类型封装        支持                           不支持

如果只是在某个业务逻辑中需要获取一下配置文件中的某项值，使用@value，在控制器中只需要获取一下值就行
如果bean中用来映射值时，用ConfigurationProperties（默认从全局配置文件中）

21、1）@PropertySource（value={“classpath：person.properties”}）
读取指定的文件，可以指定ConfigurationProperties读取的文件，加载指定的配置文件，并绑定到person变量中
2）@ImportResource：导入Spring的配置文件，让配置文件里面的内容生效
加入自己设置的配置文件beans.xml，在主运行程序上面设置ImportReourse（location={“classpath：beans.xml”}）
3）@Configuration
SpringBoot推荐全注解
加@Configuration，指明当前类是一个配置类，就是来替代之前的Spring配置文件
====@Bean
public HelloService helloService(){} 将方法的返回值添加到容器中；容器这个组件默认的id是方法名
22、占位符：
1）随机数：${random.int}
2）获取之前匹配的值：name: ${person.lastName}小狗
3）没有则可以设置默认值（加冒号）：lastName: zhangsan${app.name:123}

23、Profile：对不同环境提供不同配置功能的支持，可以通过激活、指定参数等方法切换环境
我们在配置文件编写的时候，文件名可以是application-{profile}.properties/yml
1)默认使用application.properties
例如：application.dev.properties 在application.properties设置spring.profiles.active=dev即可启动

2)在yml中可以分档设置：
    server:
      port:8088
    spring:
      profiles:
        active: dev   //激活配置模块
    ---
    server:
      port:8087

    spring:
      profiles: dev
    ---
    server:
        port:8086

    spring:
      profiles: prod
3)也可以用命令行  或者  Program arguments后面填写激活模块

24、配置文件加载位置(优先级从高到低，高的会覆盖低的，互补配置，可以通过spring.config.location来改变默认配置(适合打包后添加配置文件）：
1）项目打包后，点击左下方Termina输入用命令设置：cd target
2）再输入：java -jar spring-boot-01-0.0.1-SNAPSHOT.jar（打包后的项目名）–spring.config.location=配置文件的路径
）
- file:/config/ 项目名下面下新建config文件下面
- file:./ 项目名下面
- classpath:/config/ main/resources下新建config文件下面
- classpath:/ main/resources下面
