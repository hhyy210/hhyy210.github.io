VsCode环境搭建：

1、安装cnpm：
npm install -g cnpm –registry=https://registry.npm.taobao.org

设置淘宝镜像：npm config set registry http://registry.npm.taobao.org/

2、安装vue：cnpm install –g @vue/cli

3、创建vue项目：在创建文件的地方打开cmd：vue ui：可视化项目管理工具:自定义看视频(勾选vuex，router）

4、基本结构：
components：组件
router：路由
store：数据状态管理；所有的组件监听这个内容，达到同步
views：显示的内容
App和main.js《入口》

NPM脚本（运行调试所用）
serve：调试：可以直接点击运行，也可以在终端输入npm run serve，下方有进入的网址
build：可以直接点击运行，也可以在终端输入npm run build，loading效果、webpack编译、输出信息

在package.json里面设置

"scripts": {
  "serve": "vue-cli-service serve",
  "build": "vue-cli-service build"
},

5、安装element：
在终端：cnpm i element-ui -S
在main.js中加入依赖：
import Element from ‘element-ui’
import ‘element-ui/lib/theme-chalk/index.css’
在全局使用：
Vue.use(Element)

6、安装axios(跟ajax的地位一样)
$ npm install express –save
cnpm install axios –save
在main.js中引进：
import axios from ‘axios’
在全局使用：
Vue.prototype.$axios=axios

10、Vue Cli中public文件夹及src文件夹中assets文件夹的区别

public文件夹及assets文件夹都是用来存储静态资源的，但是在webpack处理编译是有区别的。

assets文件夹

assets文件夹是通过相对路径被引入的。这类引入会被webpack处理。

public文件夹

放在public文件夹下的资源会被直接复制，不会经过webpack的打包处理，所以如果引用的资源不希望被webpack处理时，可将其放在public文件夹中，

在index内引用资源，如果引用的资源不想被webpack处理，可以加上前缀

14、钩子函数：钩子函数是在一个事件触发的时候，在系统级捕获到了他，然后做一些操作。

15、store：数据状态管理
1、可安装store
2、在main.js里面导入并引用：
import store from ‘./store’
new Vue({
router,
store,
render: h => h(App)
}).$mount(‘#app’)
3、在src下面有个文件夹store（没有则新建），下面有一个文件index.js文件
import Vue from ‘vue’
import Vuex from ‘vuex’

    Vue.use(Vuex)

    export default new Vuex.Store({
          state: { //属性
                token: '',
                userInfo: JSON.parse(sessionStorage.getItem("userInfo"))
          },
          mutations: { //修改值的方法，get、set
                SET_TOKEN: (state, token) => {
                      state.token = token
                      //放到localstorage里面，即使浏览器关闭了也可以存入localstorage里面
                      localStorage.setItem('token', token)
                },
                SET_USERINFO: (state, userInfo) => {
                      state.userInfo = userInfo
                      sessionStorage.setItem('userInfo', JSON.stringify(userInfo)) //一次回话的结果，浏览器关闭后会丢失
                },
                REMOVE_INFO: (state) => {
                      state.token = ''
                      state.userInfo = {}
                      localStorage.setItem('token', '')
                      sessionStorage.setItem('userInfo', JSON.stringify('')) //一次回话的结果，浏览器关闭后会丢失
                }
          },
          getters: {
                getUser: state => {
                      return state.userInfo
                }
          },
          actions: {},
           modules: {}
    })
